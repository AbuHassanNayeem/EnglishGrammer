**MAXIM OF QUANTITY (GRICEAN PRAGMATICS)**
├── **Definition**
│   ├── A conversational principle stating that a speaker should provide **as much information as required**, **neither more nor less**, for the purpose of the exchange.
│   ├── Proposed by **H. P. Grice (1975)** under the **Cooperative Principle**.
│   └── Concerned with **amount of information**, not truth or relevance.
│
├── **Core Function**
│   ├── Regulates **informativeness** in communication
│   ├── Helps listeners infer **implicit meanings (implicatures)**
│   ├── Prevents **understatement and overstatement**
│   └── Widely tested in **dialogue-based MCQs** in competitive exams
│
├── **Main Types**
│   ├── **Observance of Maxim of Quantity**
│   │   ├── **Definition**
│   │   │   └── Speaker provides **exactly the required amount** of information.
│   │   │
│   │   ├── **Function**
│   │   │   └── Ensures clarity and efficiency without hidden meaning.
│   │   │
│   │   ├── **Examples**
│   │   │   ├── *A: Where do you live?*
│   │   │   └── *B: In Dhaka.*
│   │   │
│   │   ├── **Variations / Forms**
│   │   │   ├── Precise numerical answers
│   │   │   └── Direct factual responses
│   │   │
│   │   ├── **Transformations**
│   │   │   └── No implicature → literal meaning only
│   │   │
│   │   └── **Competitive Exam Tips**
│   │       └── If no hidden meaning is intended, the maxim is **observed**, not tested.
│   │
│   ├── **Flouting of Maxim of Quantity**
│   │   ├── **Definition**
│   │   │   └── Speaker **intentionally provides too little or too much information** to create implicature.
│   │   │
│   │   ├── **Function**
│   │   │   └── Generates **conversational implicature**.
│   │   │
│   │   ├── **Subtypes**
│   │   │   ├── **Under-informativeness**
│   │   │   │   ├── **Definition**: Giving less information than required.
│   │   │   │   ├── **Examples**
│   │   │   │   │   ├── *A: Did you finish the report?*
│   │   │   │   │   └── *B: I started it.*
│   │   │   │   │       → Implicature: *Not finished*
│   │   │   │   ├── **Exam Tip**
│   │   │   │   │   └── Partial answers usually imply **negation**.
│   │   │   │
│   │   │   └── **Over-informativeness**
│   │   │       ├── **Definition**: Giving more information than required.
│   │   │       ├── **Examples**
│   │   │       │   ├── *A: Is Ali honest?*
│   │   │       │   └── *B: He never stole money from his office.*
│   │   │       │       → Implicature: *Honesty is doubtful*
│   │   │       ├── **Exam Tip**
│   │   │       │   └── Extra detail often signals **suspicion or irony**.
│   │   │
│   │   ├── **Variations / Forms**
│   │   │   ├── Use of vague quantifiers
│   │   │   ├── Avoidance of direct answers
│   │   │   └── Excessive justification
│   │   │
│   │   ├── **Transformations**
│   │   │   └── Explicit statement → implicit conclusion (via inference)
│   │   │
│   │   └── **Competitive Exam Tips**
│   │       ├── If the answer feels **incomplete or excessive**, Quantity is flouted.
│   │       └── Look for **what is NOT said**.
│   │
│   ├── **Scalar Quantity Implicature**
│   │   ├── **Definition**
│   │   │   └── Implicature derived from choosing a **weaker term on an information scale**.
│   │   │
│   │   ├── **Function**
│   │   │   └── Creates implied negation of stronger alternatives.
│   │   │
│   │   ├── **Common Scales**
│   │   │   ├── all > most > many > some
│   │   │   ├── always > often > sometimes
│   │   │   └── must > should > may
│   │   │
│   │   ├── **Examples**
│   │   │   ├── *Some candidates passed the exam.*
│   │   │   │   → Implicature: *Not all passed*
│   │   │   └── *She sometimes arrives late.*
│   │   │       → Implicature: *Not always late*
│   │   │
│   │   ├── **Transformations**
│   │   │   └── *Some X* → *Not all X* (cancellable)
│   │   │
│   │   ├── **Restrictions & Exceptions**
│   │   │   ├── Implicature is **cancellable**
│   │   │   │   └── *Some students passed — in fact, all did.*
│   │   │   └── Legal/technical contexts may block implicature
│   │   │
│   │   └── **Competitive Exam Tips**
│   │       ├── “Some = not all” is a **top MCQ trap**
│   │       └── Always check for **stronger alternatives**
│   │
│   └── **Violation of Maxim of Quantity**
│       ├── **Definition**
│       │   └── Speaker secretly provides misleading amount of information to **deceive**.
│       │
│       ├── **Function**
│       │   └── Causes misunderstanding, not implicature.
│       │
│       ├── **Examples**
│       │   └── *He answered only half the truth to avoid blame.*
│       │
│       └── **Exam Tip**
│           └── Violation ≠ flouting (no shared awareness).
│
├── **Advanced / Edge Cases**
│   ├── **Cancellability Test**
│   │   └── If implied meaning can be cancelled → conversational implicature
│   │
│   ├── **Context Sensitivity**
│   │   └── What counts as “enough” varies by situation
│   │
│   ├── **Silence and Quantity**
│   │   └── Silence may imply refusal or ignorance
│   │
│   └── **Formal vs Informal Usage**
│       ├── Formal (academic, legal): prefers **over-explicitness**
│       └── Informal (conversation): tolerates vagueness
│
├── **Common Mistakes**
│   ├── Treating implicature as literal meaning
│   ├── Ignoring cancellability
│   ├── Confusing Quantity with Relation
│   └── Assuming numbers are always exact
│
├── **Comparison with Similar Structures**
│   ├── Quantity vs Quality
│   │   └── Quantity = amount | Quality = truth
│   ├── Quantity vs Manner
│   │   └── Quantity = how much | Manner = how expressed
│   ├── Implicature vs Entailment
│   │   └── Implicature cancellable, entailment not
│   └── Pragmatics vs Semantics
│
├── **British vs American Exam Preferences**
│   ├── **British Exams (IELTS, BCS, SSC)**
│   │   └── Focus on **context-based inference**
│   └── **American Exams (TOEFL, GRE)**
│       └── Focus on **logical strength of statements**
│
└── **Quick Revision Rules**
├── Quantity = **how much information**
├── Too little / too much → implicature
├── “Some” usually implies “not all”
├── Cancellable = conversational implicature
├── Extra detail often signals doubt
└── Ask: *What was expected vs what was said?*
