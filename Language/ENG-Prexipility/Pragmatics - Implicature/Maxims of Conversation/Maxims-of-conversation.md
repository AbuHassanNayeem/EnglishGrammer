**MAXIMS OF CONVERSATION (GRICEAN MAXIMS)**
├── **Definition**
│   ├── A set of **pragmatic principles** proposed by **H. P. Grice (1975)** governing how speakers cooperate to make conversation meaningful.
│   ├── They explain how **implied meaning (implicature)** arises beyond literal sentence meaning.
│   └── Core component of **Pragmatics** (meaning in context), not Syntax or Semantics.
│
├── **Core Function**
│   ├── Ensure **effective, cooperative communication**
│   ├── Help listeners **infer unstated meanings**
│   ├── Explain **indirect speech, politeness, irony, sarcasm**
│   └── Frequently tested in **IELTS, TOEFL, GRE, BCS, SSC, Banking MCQs**
│
├── **Theoretical Framework**
│   ├── **Cooperative Principle (CP)**
│   │   └── “Make your contribution such as is required, at the stage at which it occurs, by the accepted purpose of the talk exchange.”
│   │
│   └── **Relation to Implicature**
│       ├── Observing maxims → literal clarity
│       ├── Flouting maxims → conversational implicature
│       └── Violating maxims → deception / breakdown
│
├── **Main Types (Four Maxims)**
│   ├── **Maxim of Quantity**
│   │   ├── **Definition**
│   │   │   └── Provide **as much information as required**, no more, no less.
│   │   │
│   │   ├── **Core Rules**
│   │   │   ├── Do not give less information than needed
│   │   │   └── Do not give more information than needed
│   │   │
│   │   ├── **Function / Role**
│   │   │   └── Controls **informativeness** of utterances
│   │   │
│   │   ├── **Examples**
│   │   │   ├── *A: Where do you live?*
│   │   │   └── *B: In Dhaka.*
│   │   │       → Adequate information
│   │   │
│   │   │   ├── *He has three children.*
│   │   │   │   → Implicature: *Exactly three*
│   │   │   └── *Some students passed the exam.*
│   │   │       → Implicature: *Not all*
│   │   │
│   │   ├── **Variations / Forms**
│   │   │   ├── Scalar expressions (some, many, most)
│   │   │   └── Numerical precision vs vagueness
│   │   │
│   │   ├── **Transformations**
│   │   │   └── *Some students passed* → *Not all students passed* (implicature, not entailment)
│   │   │
│   │   ├── **Restrictions & Exceptions**
│   │   │   ├── Legal, academic contexts require **over-explicitness**
│   │   │   └── Friendly talk allows under-informativeness
│   │   │
│   │   ├── **Common MCQ Traps**
│   │   │   ├── Confusing implicature with factual meaning
│   │   │   └── Treating “some” as neutral
│   │   │
│   │   └── **Competitive Exam Tips**
│   │       ├── “Some ≠ all” is an **MCQ classic**
│   │       └── Over-information often signals **irony or impatience**
│   │
│   ├── **Maxim of Quality**
│   │   ├── **Definition**
│   │   │   └── Say only what you **believe to be true** and have **adequate evidence** for.
│   │   │
│   │   ├── **Core Rules**
│   │   │   ├── Do not say what you believe is false
│   │   │   └── Do not say that for which you lack evidence
│   │   │
│   │   ├── **Function / Role**
│   │   │   └── Ensures **truthfulness and reliability**
│   │   │
│   │   ├── **Examples**
│   │   │   ├── *He is a real Einstein!*
│   │   │   │   → Implicature: *Very intelligent* or *sarcastic*
│   │   │   └── *What a beautiful weather!* (during a storm)
│   │   │       → Irony via flouting Quality
│   │   │
│   │   ├── **Variations / Forms**
│   │   │   ├── Metaphor
│   │   │   ├── Hyperbole
│   │   │   └── Irony / sarcasm
│   │   │
│   │   ├── **Transformations**
│   │   │   └── Literal falsehood → implied opposite meaning
│   │   │
│   │   ├── **Restrictions & Exceptions**
│   │   │   ├── Fiction, jokes, irony deliberately flout this maxim
│   │   │   └── Exams test **recognition of flouting**, not morality
│   │   │
│   │   ├── **Common MCQ Traps**
│   │   │   ├── Taking sarcasm literally
│   │   │   └── Ignoring tone/context
│   │   │
│   │   └── **Competitive Exam Tips**
│   │       ├── If statement is “obviously false” → check for **irony**
│   │       └── Quality is most flouted for **humor & criticism**
│   │
│   ├── **Maxim of Relation (Relevance)**
│   │   ├── **Definition**
│   │   │   └── Be **relevant** to the topic of conversation.
│   │   │
│   │   ├── **Core Rule**
│   │   │   └── Do not introduce unrelated information
│   │   │
│   │   ├── **Function / Role**
│   │   │   └── Maintains **topic coherence**
│   │   │
│   │   ├── **Examples**
│   │   │   ├── *A: Are you coming to the party?*
│   │   │   └── *B: I have an exam tomorrow.*
│   │   │       → Implicature: *No*
│   │   │
│   │   ├── **Variations / Forms**
│   │   │   ├── Topic shift
│   │   │   └── Indirect refusal
│   │   │
│   │   ├── **Transformations**
│   │   │   └── Irrelevant reply → context-based inference
│   │   │
│   │   ├── **Restrictions & Exceptions**
│   │   │   ├── Politeness strategies often break direct relevance
│   │   │   └── Courtroom questioning demands strict relevance
│   │   │
│   │   ├── **Common MCQ Traps**
│   │   │   ├── Treating indirect answer as confusion
│   │   │   └── Ignoring shared situational knowledge
│   │   │
│   │   └── **Competitive Exam Tips**
│   │       ├── Relevance-based implicature is **context-heavy**
│   │       └── Often tested in dialogue-based questions
│   │
│   └── **Maxim of Manner**
│       ├── **Definition**
│       │   └── Be **clear, brief, and orderly**; avoid ambiguity and obscurity.
│       │
│       ├── **Core Rules**
│       │   ├── Avoid ambiguity
│       │   ├── Avoid unnecessary complexity
│       │   └── Be orderly
│       │
│       ├── **Function / Role**
│       │   └── Controls **how** something is said, not **what** is said
│       │
│       ├── **Examples**
│       │   ├── *The suspect was seen entering a vehicle of the two-wheeled variety.*
│       │   │   → Implicature: *Something unusual / deliberate vagueness*
│       │   └── *He passed away* instead of *He died*
│       │       → Polite obscurity
│       │
│       ├── **Variations / Forms**
│       │   ├── Euphemism
│       │   ├── Technical jargon
│       │   └── Over-formality
│       │
│       ├── **Transformations**
│       │   └── Clear form ↔ marked/complex form (implies special meaning)
│       │
│       ├── **Restrictions & Exceptions**
│       │   ├── Academic writing allows complexity
│       │   └── Everyday speech favors simplicity
│       │
│       ├── **Common MCQ Traps**
│       │   ├── Confusing Manner with Quantity
│       │   └── Ignoring wording style
│       │
│       └── **Competitive Exam Tips**
│           ├── If wording is strange → **Manner is flouted**
│           └── “How it’s said” is the key clue
│
├── **Advanced / Edge Cases**
│   ├── **Flouting vs Violating Maxims**
│   │   ├── Flouting → obvious, creates implicature
│   │   └── Violating → hidden, misleads listener
│   │
│   ├── **Multiple Maxims Flouted Together**
│   │   └── Sarcasm may flout Quality + Relation
│   │
│   ├── **Cultural Variation**
│   │   └── What counts as “enough” or “relevant” varies culturally
│   │
│   └── **Silence as Maxim Use**
│       └── Silence can imply refusal or disagreement
│
├── **Common Mistakes**
│   ├── Treating maxims as grammatical rules
│   ├── Ignoring context in interpretation
│   ├── Confusing implicature with entailment
│   └── Assuming maxims are always obeyed
│
├── **Comparison with Similar Structures**
│   ├── Maxims vs Grammar Rules
│   ├── Pragmatics vs Semantics
│   ├── Implicature vs Presupposition
│   └── Explicit meaning vs Intended meaning
│
├── **British vs American Exam Preferences**
│   ├── **British Exams (IELTS, BCS)**
│   │   └── Emphasize **contextual interpretation & pragmatics**
│   └── **American Exams (TOEFL, GRE)**
│       └── Emphasize **speaker intention & logical inference**
│
└── **Quick Revision Rules**
├── Quantity → How much is said
├── Quality → Truthfulness
├── Relation → Relevance
├── Manner → Clarity & style
├── Flouting creates implicature
└── Context decides everything
